TLA+ cheat sheet

Structure

---- MODULE X ----Every .tla file is a module. Name must match the filename.
EXTENDS IntegersImport another module. Integers gives you +, -, *, comparisons.
VARIABLES x, yDeclare state variables. A "state" is one assignment of values to all variables.
Foo ==Define an operator (like a function). Foo == x > 0 means Foo is true when x is positive.
Init ==The starting state. Constrains what values variables can begin with.
Next ==What can happen in one step. Usually a disjunction (\/) of named actions.
Spec ==The full specification. Combines Init, Next, and optionally fairness.
====End of module.

Logic

/\AND -- both sides must be true.
\/OR -- at least one side must be true.
=>Implies. "A => B" means if A then B.
~NOT.
#Not equal.
\inSet membership. x \in {1,2,3}
\notinNot in the set.
IF P THEN A ELSE BConditional expression.
CASE P -> A [] Q -> BMulti-branch conditional.

State and change

x'The value of x in the next state. x' = x + 1 means x increments.
UNCHANGED xThis action does not change x. Equivalent to x' = x.
UNCHANGED <<x, y>>Neither x nor y changes in this step.
[][Next]_varsEvery step either satisfies Next or changes nothing (stuttering is allowed).
<<a, b, c>>A tuple. Used to group variables.

The .cfg file

SPECIFICATION SpecWhich operator in the .tla defines the system. Usually "Spec".
INVARIANT FooCheck operator Foo in every reachable state. Fail = found a counterexample.
PROPERTY FooCheck temporal property Foo over entire behaviors (liveness).
CONSTRAINT FooOnly explore states where Foo is true. Limits the search space.

Temporal (advanced)

<>(P)Eventually P becomes true.
[](P)P is always true (in every state).
WF_vars(Action)Weak fairness -- if Action stays enabled, it must eventually happen.
